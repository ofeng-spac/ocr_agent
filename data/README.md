# 数据目录说明

本目录用于承载当前项目第二阶段开发所需的数据资产。

当前核心目标不是扩充模型能力，而是把“视频识别结果”和“说明书权威字段”连接起来，形成可校验、可追踪、可检索的数据基础。

## 目录结构

```text
data/
├── source_docs/
│   ├── raw/         # 原始来源文件，保留 PDF/TXT 原件
│   └── text/        # 统一转成纯文本后的版本，供字段抽取和 RAG 使用
└── structured/
    ├── drug_catalog.json      # 当前 11 个类别的标准药品目录
    └── leaflet_fields.jsonl   # 从说明书中抽出的结构化字段
```

## 当前阶段用途

本目录当前主要支撑以下开发任务：

1. 建立标准药品目录
2. 构建说明书字段抽取脚本
3. 实现标准名校验与归一
4. 为 RAG 构建受控知识源

当前字段抽取脚本为：

1. `scripts/build_leaflet_fields.py`

## 后续约束

后续新增资料时，必须遵守以下规则：

1. 原始文件先进入 `source_docs/raw/`
2. 文本化后进入 `source_docs/text/`
3. 新药品只有在进入 `drug_catalog.json` 后，才能参与标准名校验
4. 没有文档支撑的药品，不能直接作为“强确认”结果输出
