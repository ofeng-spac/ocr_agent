# 测试开发实习学习与项目补强路线图

## 1. 文档目的

本文档用于把当前项目转成一条适合“测试开发实习”投递的学习与实战路线。

目标不是继续无上限扩功能，而是围绕“测试开发”岗位要求，系统补齐：

1. 测试基础
2. 接口测试
3. 自动化回归
4. Mock / Stub
5. 前端自动化
6. CI/CD
7. Linux / Git / 环境管理
8. 数据与索引校验
9. 性能测试

后续我们按本文档逐项推进。

每完成一块，就把对应复选框改成 `[x]`。

## 2. 当前项目现状

当前项目已经具备一个可演示的 Agent 系统闭环：

1. 视频药品识别
2. 标准名校验
3. 字段检索 + chunk 检索双轨 RAG
4. Qdrant 向量数据库
5. FastAPI 后端接口
6. LangGraph 工作流轨迹
7. React 前端演示页
8. 审计日志
9. 第一层和第二层评测脚本

这意味着：

1. 项目功能层已经足够做“测试开发练手场景”
2. 现在最该补的是“如何系统地测它”

## 3. 总学习策略

整体分两条主线并行推进：

### 3.1 学习主线

把这些知识直接落到当前项目中，形成：
